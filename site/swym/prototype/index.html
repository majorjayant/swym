3 · Data contract — recommended, deliberately not built (Kano: basic)

Biggest finding of the whole exercise — the export pipeline itself is broken. These go to the source system team as suggestions, each one traced to a computed fact:

RecommendationEvidence
Emit library_card_id / device_session_id on interaction events, as the system's own description claims0 of 470 A rows carry either field
Rename or fix terminal_session_id — it is not a session86/150 groups span >1 branch; 96/150 span >1 day; median span ≈16 days
Emit branch_terminal_id on every event, not 25.5%350/470 rows lack it — a hard ceiling on any terminal-based linkage
Log checkout events in the interaction streamno checkout event type exists; checkouts survive only as per-session lists (22 occurrences)
Add validation assertions at export time (uniqueness, fill rates, session coherence)the full assertion set is this repo's analysis/ scripts, reusable as-is
If pre-login linkage is ever wanted: the deterministic rule exists (same terminal, login within 45 min)120/120 events resolve uniquely, zero ambiguity, zero conflicts — deferred for privacy/policy review, not feasibility
